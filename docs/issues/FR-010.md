# [FR-010] Cooldown, Queue & Startup

## Labels
`MUS`, `polish`, `phase-5`

## User Story
As a streamer, I want a cooldown period after Chuks speaks and a startup script that verifies all dependencies, so that the system is reliable.

## Proposed Solution

### Overview
Add cooldown logic and a small trigger queue to the orchestrator. Create a startup script that checks Kokoro, VB-Cable, and mic availability before launching.

### Implementation Flow
1. Add cooldown timer to `orchestrator.py` — block new triggers for N seconds after speech
2. Add trigger queue (max 2 pending) — discard excess triggers during cooldown
3. Create `scripts/start.py` — dependency checker + launcher
4. Add proper `logging` throughout all modules
5. Add error recovery: retry failed API calls, skip on persistent failure

### Key Considerations
- Cooldown default: 15 seconds (configurable via `.env`)
- Queue holds max 2 triggers during cooldown, discards rest
- Startup script should: ping Kokoro, list audio devices, verify `.env` values exist
- Use `logging` module with `INFO` level for normal ops, `ERROR` for failures

## Acceptance Criteria

- [ ] AI does not respond more than once per cooldown period
- [ ] Max 2 triggers queued during cooldown, excess discarded
- [ ] `scripts/start.py` checks Kokoro at `:8880` and warns if down
- [ ] Startup shows available audio devices and selected indices
- [ ] System runs for 10+ minutes without crashing
- [ ] All errors logged cleanly, no unhandled exceptions
