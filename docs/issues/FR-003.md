# [FR-003] Kokoro TTS Integration

## Labels
`MUS`, `audio`, `phase-2`

## User Story
As a streamer, I want the AI's text responses converted to natural speech, so that Chuks can speak aloud on stream.

## Proposed Solution

### Overview
Build a TTS wrapper that sends text to the local Kokoro FastAPI Docker container and receives audio bytes. Wire it with an audio playback utility that outputs to VB-Cable.

### Implementation Flow
1. Ensure Kokoro Docker is running: `docker run -p 8880:8880 ghcr.io/remsky/kokoro-fastapi-cpu:latest`
2. Create `src/tts.py` — POST text to Kokoro, receive audio bytes
3. Create `src/audio.py` — play audio buffer to a specific output device (VB-Cable)
4. Create `scripts/list_devices.py` — utility to discover audio device indices
5. Wire into orchestrator: LLM response → TTS → audio playback

### Technical Approach

```python
import httpx
import sounddevice as sd
import numpy as np

async def text_to_speech(text: str) -> bytes:
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"{KOKORO_BASE_URL}/v1/audio/speech",
            json={"input": text, "voice": KOKORO_VOICE, "response_format": "pcm"},
        )
        return response.content

def play_audio(audio_bytes: bytes, device_index: int):
    audio = np.frombuffer(audio_bytes, dtype=np.int16).astype(np.float32) / 32768.0
    sd.play(audio, samplerate=24000, device=device_index)
    sd.wait()
```

### Key Considerations
- Use `pcm` response format for direct playback (no decoding needed)
- `sd.play()` is blocking — must run in a thread pool executor from async context
- VB-Cable device index must be set in `.env`
- Test audio output to regular speakers first, then switch to VB-Cable
