# [FR-001] Core LLM Chat Loop

## Labels
`MUS`, `core`, `phase-1`

## User Story
As a streamer, I want to send text to the AI and get personality-driven responses, so that Chuks can converse.

## Proposed Solution

### Overview
Build a Groq LLM wrapper and a FastAPI endpoint that accepts text, assembles a context-rich prompt, and returns Chuks' response.

### Implementation Flow
1. Create `src/llm.py` — async Groq chat completion wrapper
2. Create `config/persona.json` — Chuks' personality config
3. Create `src/main.py` — FastAPI app with `/chat` POST endpoint
4. Wire: incoming text → persona + history → Groq → response text

### Technical Approach

```python
from groq import AsyncGroq

async def chat(text: str, history: list[dict]) -> str:
    client = AsyncGroq()
    messages = [{"role": "system", "content": persona_prompt}] + history + [{"role": "user", "content": text}]
    response = await client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=messages,
        max_tokens=150,
        temperature=0.8,
    )
    return response.choices[0].message.content
```

### Key Considerations
- Max 150 tokens per response to keep Chuks concise (40 words ~ 50-60 tokens)
- History is a rolling list of the last 30 messages
- System prompt assembled from `persona.json` fields

## Acceptance Criteria

- [ ] `persona.json` exists with name, personality, style, and rules
- [ ] POST to `/chat` with `{"text": "Hello"}` returns personality-appropriate response
- [ ] History is maintained across multiple requests in the same session
- [ ] Response respects persona rules (e.g., max length, no hallucination)
