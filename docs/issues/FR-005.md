# [FR-005] Groq Whisper STT + Mic Capture

## Labels
`MUS`, `audio`, `stt`, `phase-3`

## User Story
As a streamer, I want my mic audio transcribed in real-time via Groq's Whisper API, so that Chuks can hear me.

## Proposed Solution

### Overview
Capture mic audio in a background thread using `sounddevice`, detect speech/silence boundaries, send speech chunks to Groq Whisper API, and emit transcript text to the orchestrator.

### Implementation Flow
1. Create `src/stt.py` — mic capture + silence detection + Groq Whisper wrapper
2. Background thread records audio continuously
3. Silence detection (RMS threshold) identifies speech boundaries
4. When speech ends (silence > 1.5s), send the audio chunk to Groq Whisper API
5. Alternate between `whisper-large-v3` and `whisper-large-v3-turbo` per request (FR-006)
6. Emit transcript text to orchestrator via callback or async queue

### Technical Approach

```python
import sounddevice as sd
import numpy as np
from groq import Groq
import io, wave

WHISPER_MODELS = ["whisper-large-v3", "whisper-large-v3-turbo"]
model_index = 0

def get_next_model() -> str:
    global model_index
    model = WHISPER_MODELS[model_index % len(WHISPER_MODELS)]
    model_index += 1
    return model

def is_silent(audio_chunk: np.ndarray, threshold: float = 0.01) -> bool:
    rms = np.sqrt(np.mean(audio_chunk ** 2))
    return rms < threshold

def transcribe(audio_bytes: bytes) -> str:
    client = Groq()
    result = client.audio.transcriptions.create(
        file=("audio.wav", audio_bytes),
        model=get_next_model(),
        language="en",
    )
    return result.text
```

### Key Considerations
- Mic capture at 16000 Hz, mono, int16 — Whisper's expected format
- Silence threshold (~0.01 RMS) needs tuning per mic
- Model rotation doubles effective rate limit (40 req/min total)
- Audio must be WAV-encoded before sending to Groq (use `io.BytesIO` + `wave` module)
- Buffer management: keep a rolling window, only send when speech detected + silence follows
