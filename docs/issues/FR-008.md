# [FR-008] OBS Avatar Overlay

## Labels
`MUS`, `overlay`, `phase-4`

## User Story
As a streamer, I want an animated SVG avatar in OBS that reacts when Chuks is idle, thinking, or talking, so that viewers have a visual representation of the AI co-host.

## Proposed Solution

### Overview
Build a standalone HTML page with an SVG avatar, CSS animations for each state, and a WebSocket client that listens for state changes from the orchestrator.

### Implementation Flow
1. Create `overlay/index.html` — SVG avatar with transparent background
2. Create `overlay/styles.css` — keyframe animations for idle, thinking, talking states
3. Create `overlay/avatar.js` — WebSocket client that toggles CSS classes
4. Add WebSocket endpoint to `src/main.py` at `/ws/avatar`
5. Orchestrator sends state messages: `{"state": "idle|thinking|talking"}`
6. OBS Browser Source pointed at `http://localhost:8000/overlay/index.html`

### Technical Approach

```html
<!-- SVG avatar with state classes -->
<div id="avatar" class="idle">
  <svg><!-- Avatar SVG here --></svg>
</div>
```

```css
.idle svg { animation: breathe 3s ease-in-out infinite; }
.thinking svg { animation: pulse 0.5s ease-in-out infinite; }
.talking svg { animation: talk 0.15s steps(2) infinite; }
```

```javascript
const ws = new WebSocket("ws://localhost:8000/ws/avatar");
ws.onmessage = (event) => {
  const { state } = JSON.parse(event.data);
  document.getElementById("avatar").className = state;
};
```

### Key Considerations
- Transparent background is critical for OBS compositing
- SVG should be simple enough for CSS animations (circles, shapes)
- WebSocket should auto-reconnect on disconnect
- Avatar design can be upgraded later (Rive, PNGtuber) without changing the WebSocket protocol

## Acceptance Criteria

- [ ] Overlay page renders at `http://localhost:8000/overlay/index.html`
- [ ] Background is truly transparent (no white/black)
- [ ] Idle animation plays by default (subtle breathing)
- [ ] Thinking animation triggers on LLM call start
- [ ] Talking animation triggers during audio playback
- [ ] Returns to idle after speech completes
- [ ] Works in OBS Browser Source
